# 🧠 Advanced Agentic Document Intelligence RAG (Production-Ready)

Быстрый, стабильный и невероятно точный корпоративный поисковик по документам на базе **Advanced Agentic RAG** (Query Expansion + CRAG + Самопроверка). Сервис позволяет загружать документы (PDF, Word, TXT, MD), интеллектуально дробить их на смысловые блоки и задавать вопросы естественным языком, получая развернутые ответы с обязательными ссылками на источники.

## ✨ Ключевые особенности и архитектура
- 🤖 **Agentic LangGraph Pipeline:** Запрос проходит через интеллектуальный граф с обратной связью: `rewrite → retrieve → grade → generate ↺ self_check`. 
- 🔀 **Multi-Query Expansion:** Перед поиском LLM переписывает исходный запрос пользователя, создавая 2 дополнительные синонимичные формулировки. Это радикально повышает полноту поиска (Recall), если пользователь использовал нестандартные термины.
- ⚖️ **CRAG (Corrective RAG) & LLM Grader:** Найденные фрагменты текста проходят строгую проверку через независимый LLM-узел (Grader). Мусорные и нерелевантные куски отсекаются *до* того, как попадут в финальную генерацию.
- 🧩 **Умный семантический чанкинг:** Текст больше не режется вслепую. Используется `RecursiveCharacterTextSplitter` из LangChain, который грамотно делит документы по абзацам и предложениям, сохраняя целостность смысла.
- 🎯 **Chain-of-Thought (CoT) & XML Разметка:** Внутренний контекст оборачивается в теги `<document>`. Модель принудительно делает блок `**Анализ:**` перед тем, как выдать финальный `**Ответ:**`. Это сводит риск галлюцинаций практически к нулю.
- 🛡 **Self-RAG & Самоисправление:** Встроенный узел `self_check` (LLM-as-a-Judge) проверяет каждый ответ на выдуманные факты. При обнаружении галлюцинации граф делает петлю и переписывает ответ (до 3-х попыток).
- **Global RRF (Reciprocal Rank Fusion):** Пакетное слияние результатов векторного (`NVIDIA E5-v5`) и лексического (`BM25`) поиска для *всех* сгенерированных запросов.
- ⚡ **Bypass-режим (Прямой чат):** Возможность общения с LLM напрямую без RAG-контекста. Переключается чекбоксом в левой панели. Поддерживает историю разговора (до 10 сообщений).

## 🛠 Технологический стек
* **Backend:** Python 3.10+, FastAPI, Uvicorn
* **RAG Pipeline:** LangGraph (StateGraph с циклами), LangChain Core, LangChain Text Splitters
* **Vector DB:** ChromaDB (локальная персистентная база)
* **Keyword Search:** rank-bm25 (в оперативной памяти)
* **LLM & Embeddings:** OpenAI Async Client (совместимый с NVIDIA NIM)
* **Resilience:** tenacity (exponential backoff для API-вызовов)
* **Frontend:** Vanilla HTML5, CSS3, JavaScript (mobile-friendly, Toasts, без тяжелых фреймворков)

## 💾 Перенос и резервное копирование базы данных

ChromaDB хранит все проиндексированные документы в обычной папке на диске (`chroma_data/` в корне проекта). Это значит:

- **Перенос на другой ПК:** Просто скопируйте папку `chroma_data/` в корень проекта на новой машине — все документы и индексы сохранятся полностью. Переиндексировать ничего не нужно.
- **Резервная копия:** Достаточно заархивировать `chroma_data/` (например, `tar -czf chroma_backup.tar.gz chroma_data/`). Для восстановления — распакуйте обратно.
- **⚠️ Важно:** `BM25`-индекс хранится **только в оперативной памяти** и не сохраняется на диск. После перезапуска сервера он автоматически и очень быстро пересобирается из данных ChromaDB.

### 2. Установка зависимостей
Рекомендуется использовать виртуальное окружение:
```bash
python -m venv venv
source venv/bin/activate  # Для Linux/Mac
# venv\Scripts\activate   # Для Windows

pip install -r requirements.txt
```

### 3. Настройка окружения (.env)
Создайте файл `.env` в корне проекта. Все настройки моделей и API вынесены в переменные окружения для удобства:
```env
# API Configuration
NVIDIA_API_KEY=nvapi-ваш-ключ-здесь
NVIDIA_BASE_URL=https://integrate.api.nvidia.com/v1

# Models
LLM_MODEL=openai/gpt-oss-20b
EMBED_MODEL=nvidia/nv-embedqa-e5-v5
BYPASS_MODEL=minimaxai/minimax-m2.1

# Feature Flags (управление качеством и стоимостью)
# Подробное описание см. в таблице ниже
ANONYMIZED_TELEMETRY=False
ENABLE_QUERY_EXPANSION=true   # Multi-Query: +2 синонимичных запроса → выше Recall
ENABLE_LLM_GRADER=true        # CRAG: фильтрация чанков LLM → выше Precision
ENABLE_SELF_CHECK=true        # Self-RAG: проверка на галлюцинации → ниже риск выдумок
```

**Описание переменных моделей:**
- `NVIDIA_API_KEY` — обязательный ключ для LLM и Embeddings
- `NVIDIA_BASE_URL` — API-эндпоинт (по умолчанию: NVIDIA NIM)
- `LLM_MODEL` — модель для RAG-генерации
- `EMBED_MODEL` — модель для эмбеддингов
- `BYPASS_MODEL` — модель для прямого чата (без RAG)

#### Feature Flags (управление качеством и стоимостью)

| Параметр | Описание | Влияние на качество | Влияние на стоимость |
|----------|----------|---------------------|----------------------|
| `ENABLE_QUERY_EXPANSION` | **Multi-Query Expansion** — перед поиском LLM генерирует 2 альтернативные формулировки запроса (синонимы, аббревиатуры) | ⬆️ **Значительно повышает** полноту поиска (Recall), особенно для специфических терминов | +1 запрос к LLM на каждый вопрос |
| `ENABLE_LLM_GRADER` | **CRAG LLM Grader** — каждый найденный чанк проверяется LLM на релевантность ("Содержит ли ответ?") | ⬆️ **Повышает** точность, отсеивая мусорный контекст до генерации | +1 запрос на каждый чанк (до 4-5 чанков) |
| `ENABLE_SELF_CHECK` | **Self-RAG валидация** — финальный ответ проверяется LLM на галлюцинации | ⬆️ **Снижает** риск галлюцинаций, при обнаружении делает retry | +1 запрос к LLM на каждый ответ |

**Рекомендации:**
- Для **максимального качества** — все флаги включены (`true`)
- Для **экономии токенов** — отключайте `ENABLE_SELF_CHECK` (минимальная потеря качества)
- Для **быстрого тестирования** — отключите все флаги, оставьте только базовый поиск

### 4. Запуск сервера

uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload

После запуска откройте в браузере: **http://localhost:8000**

## 📄 Рекомендации по форматам документов

Качество ответов нейросети напрямую зависит от извлеченного текста. С новым сплиттером (`RecursiveCharacterTextSplitter`) система стала гораздо умнее, но правила чистоты текста сохраняются:

1. 🥇 **Markdown (`.md`) и Plain Text (`.txt`) — Идеально.** Алгоритм идеально бьет его на абзацы, сохраняя структуру заголовков и списков.
2. 🥈 **Microsoft Word (`.docx`) — Хорошо.** Успешно извлекает абзацы и корректно читает простые таблицы, склеивая строки через `|`.
3. 🥉 **PDF (`.pdf`) — Удовлетворительно.** Извлечение текста может сбиваться из-за колонтитулов или водяных знаков. Не распознает картинки-сканы без OCR-слоя.

## 🛠 Под капотом (Архитектура Advanced RAG)

Наш конвейер обработки каждого вопроса — это направленный граф, который принимает решения на лету:

1. **`rewrite` (Multi-Query)**:
   - LLM анализирует вопрос пользователя и генерирует 2 альтернативных варианта (например, переводит аббревиатуры или подбирает синонимы).
2. **`retrieve` (Пакетный Гибридный поиск)**:
   - Векторный поиск (семантика) и BM25 (лексика) запускаются параллельно сразу для **всех 3-х вариантов запроса**.
   - Сотни полученных результатов математически сливаются с помощью алгоритма Global RRF (Reciprocal Rank Fusion).
3. **`grade` (CRAG Фильтрация)**:
   - Сначала быстрый отсев по косинусному расстоянию (score).
   - ⚡ Затем лучшие чанки асинхронно отправляются в LLM (Grader) с вопросом: *"Есть ли тут ответ?"*. Если LLM отвечает `NO`, чанк удаляется из контекста.
   - **Fast-Fail маршрутизация:** Если контекста не осталось, система сразу возвращает ответ «Информации нет», обходя генератор.
4. **`generate` (CoT Генерация)**:
   - Текст оборачивается в XML-теги `<document source="...">`.
   - Запускается механизм Chain-of-Thought — модель сначала пишет рассуждение `**Анализ:**`, а только потом формирует финальный ответ.
5. **`self_check` (Валидация)**:
   - Ответ и контекст отправляются независимому LLM-судье на выявление галлюцинаций.
   - 🔄 **Self-Correction:** Если факт выдуман, граф делает петлю назад в генератор с усиленным, строгим промптом. При успехе — отдаёт результат пользователю.

## ⚡ Bypass-режим (Прямой чат)

Помимо RAG-поиска по документам, приложение поддерживает **режим прямого общения с LLM** без использования загруженных документов.

### Как использовать
1. В левой панели найдите переключатель **«Прямой чат (без RAG)»**
2. Активируйте чекбокс — интерфейс переключится в режим bypass
3. Общайтесь с ИИ напрямую

### Особенности bypass-режима
- 🤖 Использует отдельную модель (`BYPASS_MODEL` из `.env`, по умолчанию: `minimaxai/minimax-m2.1`)
- 💬 Поддерживает историю разговора (последние 10 сообщений)
- ⚠️ Не использует RAG-контекст — модель не видит загруженные документы
- 🧹 Автоматически очищает ответы от внутренних рассуждений модели (thought chain)

### Когда использовать
- Для общих вопросов без привязки к документам
- Для brainstorming и генерации идей
- Для тестирования различных моделей
