# 🧠 Agentic Document Intelligence RAG (Production-Ready)

Быстрый, стабильный и точный корпоративный поисковик по документам на базе **Agentic Hybrid RAG** (векторный поиск + лексический поиск + самопроверка). Сервис позволяет загружать документы (PDF, Word, TXT, MD), индексировать их и задавать вопросы естественным языком с получением точных ответов и ссылок на источники.

## ✨ Ключевые особенности и архитектура
- 🤖 **Agentic LangGraph Pipeline (Новое):** Запрос проходит не по прямой трубе, а через интеллектуальный граф с обратной связью: `retrieve → grade → generate ↺ self_check`. Идеальное использование условных переходов (Conditional Edges) и циклов.
- 🛡 **Self-RAG & Самоисправление (Новое):** Встроенный узел `self_check` (LLM-as-a-Judge) проверяет каждый сгенерированный ответ на наличие галлюцинаций. Если факт выдуман, граф делает петлю назад и заставляет модель переписать ответ с более строгим промптом (до 3-х попыток).
- ⚡ **Fast-Fail маршрутизация (Новое):** Если после фильтрации документов не осталось релевантных фрагментов, граф мгновенно прерывает работу, не тратя токены и время на бесполезную генерацию.
- 🚀 **Высокая производительность:** Параллельное выполнение векторного (I/O) и BM25 (CPU) поиска через `asyncio.gather`. Ускоренная сортировка тысяч чанков с помощью `heapq`.
- 📱 **Адаптивный UI/UX:** Интерфейс полностью оптимизирован для смартфонов и планшетов (выезжающий сайдбар). Плавные анимации через `requestAnimationFrame`, стэкающаяся система Toasts без состояния гонки.
- **Гибридный поиск (Hybrid Retrieval):** Комбинация семантического поиска (`NVIDIA E5-v5`) и лексического по ключевым словам (`BM25`) с математическим слиянием **Reciprocal Rank Fusion (RRF)**.
- **Grade-фильтрация:** Нерелевантные чанки отсеиваются до отправки в LLM, жестко лимитируется количество чанков из одного документа (защита от монополизации контекста).
- **Оптимизированная память:** Устранена утечка памяти при удалении объемных документов из ChromaDB; автоочистка неиндексированных «сиротских» файлов (TTL).

## 🛠 Технологический стек
*- **Backend:** Python 3.10+, FastAPI, Uvicorn
- **Модульная архитектура (Новое):** Код разделен на пакет `rag_engine` (логика, конфиги, обработка текста) и точку входа `main.py`.
- **RAG Pipeline:** LangGraph (StateGraph с циклами и Conditional Edges), LangChain Core
* **Vector DB:** ChromaDB (локальная персистентная база)
* **Keyword Search:** rank-bm25 (в оперативной памяти)
* **LLM & Embeddings:** OpenAI Async Client (совместимый с NVIDIA NIM)
* **Resilience:** tenacity (exponential backoff для API-вызовов)
* **Frontend:** Vanilla HTML5, CSS3, JavaScript (Без тяжелых фреймворков, нативных CSS-переменных, полностью mobile-friendly)

## 💾 Перенос и резервное копирование базы данных

ChromaDB хранит все проиндексированные документы в обычной папке на диске (`chroma_data/` в корне проекта). Это значит:

- **Перенос на другой ПК:** Просто скопируйте папку `chroma_data/` в корень проекта на новой машине — все документы и индексы сохранятся полностью. Переиндексировать ничего не нужно.
- **Резервная копия:** Достаточно заархивировать `chroma_data/` (например, `tar -czf chroma_backup.tar.gz chroma_data/`). Для восстановления — распакуйте обратно.
- **⚠️ Важно:** `BM25`-индекс хранится **только в оперативной памяти** и не сохраняется на диск. После перезапуска сервера он автоматически и очень быстро пересобирается из данных ChromaDB.

```text
project_root/
└── chroma_data/       ← вся база здесь, просто скопируйте эту папку
    ├── chroma.sqlite3
    └── ...
```

```env

```
*(Если ключ не задан, сервер запустится, но выдаст ошибку при попытке обращения к LLM/Embeddings).*

### 4. Запуск сервера
```bash
uvicorn app.main:app --host 0.0.0.0 --port 8000 --reload
```
После запуска откройте в браузере: **http://localhost:8000**

## 📄 Рекомендации по форматам (Как получить лучшие ответы)

Качество ответов нейросети (RAG) **напрямую зависит от того, в каком виде загружены документы**. Нейросеть не видит картинки или сложное форматирование — она «читает» извлеченный голый текст.

### 🏆 Рейтинг форматов
1. 🥇 **Markdown (`.md`) и Plain Text (`.txt`) — Идеально.**  
   Текст максимально чистый. Алгоритм идеально бьет его на абзацы, нейросеть прекрасно понимает структуру (особенно если используются заголовки `#` и списки `-`).
2. 🥈 **Microsoft Word (`.docx`) — Хорошо.**  
   Система успешно извлекает абзацы и корректно читает простые таблицы, склеивая строки через `|`.
3. 🥉 **PDF (`.pdf`) — Удовлетворительно, но с нюансами.**  
   PDF — это визуальный формат. Извлечение текста (используется `pdfplumber` с фоллбеком на оптимизированный `pypdf`) может сбиваться из-за колонтитулов или водяных знаков. Не распознает картинки-сканы без OCR-слоя.

### 💡 Лайфхаки для базы знаний:
* **Явный контекст:** Местоимения ломают поиск. Нейросеть ищет фрагментами.
  * ❌ *Плохо:* «Договор расторгается при следующих условиях...»
  * ✅ *Хорошо:* «Договор аренды складов ООО "Ромашка" расторгается при следующих условиях...»
* **Упрощайте таблицы:** Сложные таблицы с объединенными ячейками в PDF превратятся в кашу. Используйте простые плоские таблицы `Строка -> Столбец`. Фронтенд умеет красиво рендерить Markdown-таблицы в ответах.

## 🛠 Под капотом (Архитектура Agentic RAG)

Система базируется на направленном циклическом графе (LangGraph), который принимает решения на лету:

1. **`retrieve` (Гибридный поиск)**:
   - Векторный поиск (семантика) и BM25 (по ключевым словам) запускаются асинхронно параллельно.
   - BM25 использует `heapq` для мгновенной сортировки десятков тысяч чанков.
   - Reciprocal Rank Fusion (RRF) объединяет результаты в единый пул.

2. **`grade` (Фильтрация)**:
   - Отсев «мусорных» чанков по порогу косинусного сходства. Дедупликация (`MAX_CHUNKS_PER_FILE`).
   - ⚡ **Условный переход (Fast-Fail):** Если качественных чанков не осталось, система сразу возвращает ответ «Информации нет», обходя вызов генератора.

3. **`generate` (Генерация ответа)**:
   - Формирование контекста с динамическим системным промптом (если это повторная попытка, промпт становится гораздо строже).
   - Расчет уверенности (`confidence`) на основе векторного расстояния.

4. **`self_check` (Валидация галлюцинаций)**:
   - Сгенерированный ответ и мини-версия контекста отправляются независимому LLM-валидатору.
   - Валидатор проверяет: *"Основан ли ответ строго на контексте?"*
   - 🔄 **Условный переход (Self-Correction):** Если найдена галлюцинация, граф увеличивает счетчик `attempts` и возвращает запрос в узел `generate`. При успехе (или по исчерпанию попыток) — отдаёт результат пользователю.

### 🔒 Согласованность данных и защиты
- **Автоматический Retry:** Вызовы к API обернуты в `tenacity` (до 4 попыток с экспоненциальной задержкой при ошибках сети или 429 Rate Limit).
- **Изоляция при индексации:** Глобальный `asyncio.Semaphore(5)` защищает API эмбеддингов от перегрузки при загрузке больших файлов, а `asyncio.Lock` на уровне `file_id` исключает состояние гонки (дублирование файлов).
